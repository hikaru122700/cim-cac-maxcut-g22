## 結論

ラウンド数が真に支配しているのは個別パラメータではなく、次の 2 つの設計変数:

- **ramp 速度** `dg₀²/dk ∝ L²·dP` ← L と dP が縮退した自由度
- **閾値** `g₀_th = −ln(η)` ← loss_dB が単独で決定

Optuna は num_rounds ごとに、この 2 つを別々の値に設定する 5 パラの組み合わせを発見:

- **短時間条件** (例: num_rounds=30): ramp 速度を上げ、閾値を下げ、強結合 + 強飽和で**強引に解を出す**
- **長時間条件** (例: num_rounds=10000): ramp 速度を下げ、閾値を上げ、弱結合 + 緩飽和で**ゆっくり adiabatic に到達**
- **中間 (300/1500)**: 総ラウンドが k_th より小さい場合は subthreshold で運用しきって最後だけ閾値付近を掠める戦略

> **将来のチューニング設計に向けたメモ**: L と dP は L²·dP の積で縮退しているので、`L_eff = L²·dP` の 1 変数化で探索効率が上がる可能性が高い。fANOVA 重要度で L と dP の個別寄与が小さく出るなら根拠あり。
