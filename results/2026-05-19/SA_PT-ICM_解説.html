2SAの基本原理 — 金属焼きなましのアナロジー

物理のアナロジー:
金属を高温に加熱すると原子が自由に動き回る。ゆっくり冷却すると原子は最も安定な結晶構造（エネルギー最小）に落ち着く。急冷すると欠陥だらけの脆い金属になる。

これを最適化問題に応用したのがシミュレーテッドアニーリング (SA):

高温 (T=高) 低温 (T≈0) ↓ ↓ ┌─────────────┐ ┌─────────────┐ │ ● ● ● │ 解がランダムに │ │ │ ● ● ● │ 飛び回る │ ● │ 最良解付近に │ ● ● │ (悪化も許容) │ │ 収束 └─────────────┘ └─────────────┘ エネルギー地形: ╱╲ ╱╲ ╱╲ ╱╲ ╱ ╲ ╱ ╲ 高温: どこでも ╱ ╲ ╱ ╲ ╱ ╲╱ ╲ 行ける ╱ ╲╱ ╲ ╱ ╲ ╱ ● ╲ 低温: 谷底に ╱ ╲ ╱ ╲ 閉じ込められる
核心: 高温では「悪い方向への移動」も確率的に許可することで、局所最適の谷から脱出できる。温度を下げるにつれて悪化を許さなくなり、最終的に良い解に落ち着く。

3アルゴリズムの核心 — Metropolis基準

SAの心臓部はMetropolis基準と呼ばれる受理判定:

現在の解: x,  エネルギー: E(x)
提案された新解: x',  エネルギー: E(x')

ΔE = E(x') - E(x)  (エネルギー変化量)

【受理判定】
if ΔE < 0:
    # 改善 → 常に受理
    x ← x'
else:
    # 悪化 → 確率 exp(-ΔE/T) で受理
    if random() < exp(-ΔE/T):
        x ← x'

直感的な理解

状況 高温 (T=10) 低温 (T=0.1)
ΔE = +1 の悪化 exp(-1/10) = 90% で受理 exp(-1/0.1) = 0.005% で受理
ΔE = +5 の悪化 exp(-5/10) = 60% で受理 exp(-5/0.1) ≈ 0% で受理
高温では「ちょっと悪くなってもOK」と寛容。低温では「絶対に悪化させない」と厳格。この性格の変化が、広い探索→精密な収束の流れを作る。

4冷却スケジュール — 温度の下げ方

温度 T(t) の下げ方を冷却スケジュールと呼ぶ。主に3種類:

種類 式 特徴
線形冷却 T(t) = T₀ - αt 単純だが収束が遅い
指数冷却 T(t) = T₀ · αᵗ (0 < α < 1) 実用で最も一般的
対数冷却 T(t) = T₀ / log(1+t) 理論的に最適だが極端に遅い

指数冷却を採用。T = T_start × exp(log(T_end/T_start) × progress)

5SAの限界 — なぜ単純SAでは不十分か

標準SAには構造的な弱点がある:

弱点1: 1-flipの局所性

MAX-CUT での 1-flip: A ─── B A ─── B │ │ → │ │ C ─── D C ─── D 頂点Aだけ反転 → 隣接辺3本の寄与が変わる 問題: 「A,B,Cを同時に反転すれば大改善」という 大域的な構造変化を見つけられない

弱点2: 温度の単一性

1つの温度しか持たないため、「高温での広い探索」と「低温での精密収束」を同時にできない。

弱点3: 局所最適トラップ

エネルギー地形: ╱╲ ╱ ╲ ╱╲ ╱ ╲ ╱ ╲ ╱ ● ╲____╱ ╲ ╱ 局所 大域 ╲ ╱ 最適 最適 ╲ 低温になると局所最適から抜け出せず、 大域最適に到達できない
G22での結果: 単純SAは平均13224.8、最良13314（既知最良13359に対して45不足）。CIM (13275.3) やCAC (13284.8) に劣る。

6PT-ICMとは — 最強の古典ベンチマーク

PT-ICM = Parallel Tempering + Isoenergetic Cluster Moves

SAの弱点を2つの強力な拡張で克服:

拡張1: Parallel Tempering (レプリカ交換)

【複数温度のレプリカを並列実行】 温度 T₁=10.0 ●────────●────────● 高温レプリカ (広い探索) ↕ 交換 ↕ 交換 温度 T₂=5.0 ●────────●────────● ↕ 交換 ↕ 交換 温度 T₃=1.0 ●────────●────────● ↕ 交換 ↕ 交換 温度 T₄=0.1 ●────────●────────● 低温レプリカ (精密収束) → 時間軸 → 【レプリカ交換の仕組み】 隣接温度間で、Metropolis基準に従って「解を入れ替え」 低温で良い解を見つけたら、高温に送って別経路を探索 高温で見つけた有望領域を、低温に送って精密化
アナロジー: 登山隊が複数チームに分かれ、高所チーム（大胆に動く）と低所チーム（慎重に探す）が無線で情報共有。有望なルートが見つかったらチーム間でポジションを交換。

拡張2: Isoenergetic Cluster Moves (ICM)

【クラスター単位の非局所移動】 通常SA: 1頂点ずつ反転 ○─○─●─○ │ │ │ │ ●─○─○─● ↓ ○─○─●─○ (Aだけ反転) │ │ │ │ ○─○─○─● ICM: クラスター全体を一度に反転 ○─○─●─○ │ │ │ │ ●─○─○─● ↓ ●─●─○─● (接続したクラスターを一括反転) │ │ │ │ ○─●─●─○ 【Houdayerクラスターアルゴリズム】 1. 同じ温度の2つのレプリカを選ぶ 2. 両者で「スピンが異なる」頂点の部分グラフを構築 3. そのグラフ上でランダムなクラスターを生成 4. クラスター内のスピンを2レプリカ間で交換 → エネルギーが変わらない（isoenergetic）ので常に受理！
ICMの威力:
- 1-flipでは到達できない「大域的な構造変化」を実現
- エネルギー不変なので100%受理（Metropolis判定不要）
- システムサイズが大きいほど効果が増大

PT-ICMの全体フロー

【PT-ICMアルゴリズム】

初期化:
  - M個の温度 T₁ > T₂ > ... > Tₘ を設定
  - 各温度に複数のICMレプリカを配置 (例: 温度あたり4個)

メインループ:
  for step in 1 to N_steps:

    1. 局所更新 (標準SA)
       各レプリカで K回の 1-flip Metropolis 更新

    2. レプリカ交換 (PT)
       隣接温度ペアを選び、Metropolis基準で解を交換
       P_swap = min(1, exp((β₁-β₂)(E₁-E₂)))

    3. クラスター移動 (ICM)
       同じ温度の2レプリカペアを選び、
       Houdayerクラスター交換を実行

最終出力:
  最低温度レプリカの最良解

7なぜPT-ICMがCIM比較の主流か

理由1: 古典アルゴリズムの最高性能

PT-ICMは、スピングラスやMAX-CUT問題において古典的手法の中で最も強力とされる。量子コンピュータやCIMの「優位性」を示すには、最強の古典手法と比較する必要がある。

理由2: 学術的スタンダード

論文/プロジェクト PT-ICMの使用
D-Wave Systems (量子アニーラ) 性能比較のベンチマーク標準
Google Quantum AI 量子優位性検証の古典対照
CIM研究 (Stanford, NTT) 光イジングマシンの性能評価

理由3: 公平な比較のため

「新しいスポーツカーの性能を測るなら、軽自動車ではなくF1マシンと比較すべき」

同様に、CIMの真の性能を評価するには、単純SAではなくPT-ICMと比較する必要がある。PT-ICMに勝てれば、真に価値のある技術革新と言える。

理由4: スケーリング特性

9まとめ

SA（シミュレーテッドアニーリング）

  • 金属焼きなましに着想を得た確率的最適化手法
  • Metropolis基準で「高温→低温」の冷却により解を収束
  • シンプルで実装容易だが、1-flipの局所性が限界

PT-ICM

  • Parallel Tempering: 複数温度のレプリカ並列実行 + 交換
  • ICM: クラスター単位の非局所移動（エネルギー不変で100%受理）
  • 古典アルゴリズムの最高峰、CIM/量子計算機ベンチマークの標準
  • 「PT-ICMに勝てるか」がCIMの真価を問う試金石
CIM比較における位置づけ:
単純SAは「弱い古典ベースライン」。PT-ICMは「最強の古典チャレンジャー」。CIMがPT-ICMより高速・高精度であることを示せれば、物理実装の真の優位性が証明される。
