# SB(Simulated Bifurcation)アルゴリズム — 直感重視の解説

最終更新: 2026-05-13

参考論文:
- Goto, Tatsumura, Dixon, *"Combinatorial optimization by simulating adiabatic bifurcations in nonlinear Hamiltonian systems"*, Science Advances 5, eaav2372 (2019). — aSB
- Goto et al., *"High-performance combinatorial optimization based on classical mechanics"*, Science Advances 7, eabe7953 (2021). — bSB, dSB
- Kanao, Goto, *"Simulated bifurcation assisted by thermal fluctuation"*, arXiv:2203.08361 (2022). — HbSB, HdSB

本実装: `modules/SB.py`

## 1行で言うと

**「N 個のボールを動的に変化するポテンシャルの上で転がし、最後に各ボールがどっちの谷に落ちたかで Ising スピンを決める」** という古典力学シミュレーション。

---

## ステップ 1:何を解きたかったか

MAX-CUT(辺の重みでカット最大化)は **Ising 模型** で書き直せる:

```
E_Ising(σ) = -(1/2) Σ J_ij σ_i σ_j,    σ_i ∈ {-1, +1}
```

各スピンを `+1 か -1` に割り当てて、**この E が一番小さくなる組み合わせを見つけたい**。スピン数 N=2000 だと組み合わせは 2^2000 通り、全探索は宇宙の年齢を超えて無理。

---

## ステップ 2:SB の核心トリック ─ 「離散を連続に解す」

スピン σ ∈ {-1, +1} は離散だから探索が大変。なら **連続変数 x ∈ ℝ に置き換えて**、最後に符号で離散化しよう。

```
σ_i = sign(x_i)
```

これだけだと「どんな x にしても σ にしてしまえば良い」になって解にならない。**SB の天才的なところは、x にちゃんと「物理」を与えて、自然に解に落ちるようにしたこと**。

---

## ステップ 3:核心アナロジー — 「ポテンシャル谷が成長する」

各 x_i にひとつポテンシャル V(x_i) を考える。これを **時間とともに形を変える**:

```
時刻 t=0(分岐前):

       \         /
        \       /
         \     /
          \   /
           \_/            ← 谷は原点に1つだけ


時刻 t=T(分岐後):

   \              /
    \____      ____/
        \    /             ← 谷が2つに分裂 (双井戸)
         \__/
        ↑    ↑
       x=-1  x=+1
```

ボール(初期は原点近くにいる)は、ポテンシャルが分裂した瞬間に **左右どちらか一方の谷に落ちる**。落ちる方向がスピンになる:

- 左の谷 (x≈-1) → σ = -1
- 右の谷 (x≈+1) → σ = +1

これが **分岐 (bifurcation)** の名前の由来。

---

## ステップ 4:なぜ N 個のボールが「正解」に落ちるのか?

N 個のボールが完全に独立だと、左右はサイコロで決まり、何も解けない。**ここで結合行列 J_ij を「ボール同士をつなぐバネ」として導入する**:

```
力 on x_i = - dV/dx_i + c0 Σ_j J_ij x_j
                       ─────────────────
                       近隣のボールに引かれる力
```

- `J_ij < 0`(反強磁性): ボール i と j は **逆方向に落ちたがる**
- `J_ij > 0`(強磁性): ボール i と j は **同じ方向に落ちたがる**

分岐の瞬間に、**この「ばね力」が各ボールの落ちる方向を決める**。結果、Ising エネルギーが小さい配置に落ちやすい。

---

## ステップ 5:数式での書き下ろし (aSB の場合)

各 x_i に対する運動量 y_i も導入し、Hamilton 力学:

```
H = Σ_i [ (a0/2) y_i² + (1/4) x_i⁴ + ((a0-a(t))/2) x_i² ]   ← 個別ポテンシャル
   - (c0/2) Σ_ij J_ij x_i x_j                                ← 結合
              ↑運動エネルギー    ↑Kerr 項   ↑可変調和項
```

時間微分から運動方程式:

```
dx_i/dt =  a0 y_i                                       (位置の更新)
dy_i/dt = -[x_i² + (a0 - a(t))] x_i + c0 Σ_j J_ij x_j   (運動量の更新)
```

`a(t)` を **0 → a0 までゆっくり上げる** のが「分岐パラメータのスケジュール」。

### a(t) の役割を絵で

```
ポテンシャル形状 (a(t) を上げる効果):

a(t)=0:    _______       谷 1 つ
             \_/

a(t)=0.5 a0: ___ ___     谷が浅くなり始める
               \_/

a(t)=a0:   __     __     谷 2 つに分裂
             \_./
              ↑
            分岐!
```

物理的には「中央の調和項 `((a0-a(t))/2) x_i²` が a(t) で打ち消されていって、Kerr 項 `(1/4) x_i⁴` が支配的になる」過程。

---

## ステップ 6:時間発展を計算機で解く

連続時間 `t` を離散化(`Δt` 刻み)して、**シンプレクティック・オイラー法** で更新:

```
1. y を進める:  y_i ← y_i + Δt · [-(a0-a)·x_i + c0 J@x]    (現在の x を使う)
2. x を進める:  x_i ← x_i + Δt · a0 · y_i_new              (更新後の y を使う)
   ↑この順序が大事。逆だとエネルギー保存が崩れる
```

これを `num_steps` 回繰り返し、`a(t)` を 0 から a0 まで線形にランプアップ。

## ステップ 7:3 バリアントの違いを直感で

### aSB(2019, 原典)

```
ポテンシャル: ((a0-a)/2) x² + (1/4) x⁴
              ↑可変調和項       ↑Kerr 項(双井戸を作る)
```

- 自然な双井戸を作るために Kerr の 4 乗項を使う
- 滑らかなポテンシャルで「教科書的」だが、数値計算で Kerr 項 `x³` の乗算がやや重い

### bSB(2021, ballistic)

```
ポテンシャル: ((a0-a)/2) x² だけ + 壁拘束 |x| ≤ 1
                                  ↑x=±1 に「箱の壁」を置く
壁衝突: |x_i| > 1 になったら x_i ← sign(x_i), y_i ← 0
                                                 ↑完全非弾性反射
```

- Kerr 項を捨てて、代わりに `|x|=1` に **物理的な壁** を置く
- 計算が軽くなり、aSB より高速
- ボールは壁に **弾道的(ballistic)** に飛んでいく(滑らかな谷を転がる aSB と対照的)
- 「断熱」じゃなくなる代わりに **収束が速い**

### dSB(2021, discrete)

```
bSB と同じ式だが、結合計算だけ離散化:

dy/dt = -(a0-a) x + c0 Σ J_ij · sign(x_j)
                              ↑x_j ではなく sign(x_j) を使う
```

- 結合項で「相手の符号」だけを見る
- 符号は 0/1 切替の不連続点を含むので、**エネルギー保存則がわざと壊れる**
- これが量子トンネル効果に似た「壁を乗り越える振る舞い」を作り、**局所最適から脱出できる**
- 3 つの中で**最高精度**

### 直感的順位

- **速さ**: bSB > dSB > aSB (Kerr 項なし vs あり)
- **精度**: dSB > bSB > aSB (壁トンネル機構 > 単純壁 > 滑らか双井戸)
- **解析しやすさ**: aSB > bSB > dSB (純粋ハミルトニアン vs 離散化)

### 補足:HbSB / HdSB(Kanao & Goto 2022)

加熱項 `+ γ y(t_k) Δt` を壁拘束の後で加える。乱数は使わない決定論的拡張で、壁衝突で y=0 になっても再起動できるため局所最適脱出力がさらに上がる。論文の K2000 ベンチでは HbSB が最良。

## ステップ 8:アルゴリズム全体のフロー

```
入力: J (結合行列)
出力: σ ∈ {-1, +1}^N (解)

1. x, y を小さなランダム値で初期化(各 trial で独立)

2. 時刻ステップ k = 1 ... N_step:
     a) a(t) = (k / N_step) · a0           # 分岐パラメータ更新

     b) 結合計算:
          Jx = J @ x                       # aSB / bSB
          Jx = J @ sign(x)                 # dSB

     c) y を更新:
          y_i += Δt · [ -(a0-a) x_i + c0 Jx_i ]               (bSB/dSB)
          y_i += Δt · [ -(a0-a + x_i²) x_i + c0 Jx_i ]        (aSB)

     d) x を更新:
          x_i += Δt · a0 · y_i

     e) 壁拘束(bSB/dSB のみ):
          if |x_i| > 1: x_i ← sign(x_i), y_i ← 0

     f) 加熱(HbSB/HdSB のみ):
          y_i += Δt · γ · y_i_old

     g) (オプション) σ ← sign(x) でカット値計算、best 更新

3. 最終的に σ_i = sign(x_i) を解として出力
```

## ステップ 9:なぜ「物理シミュレータ」より速くて精度も良いのか

**並列性が異次元**:

```
SA (シミュレーテッドアニーリング):
   1 ステップで 1 個のスピンを反転 → 順次処理しか無理
   ↓
   GPU/FPGA 並列化が効きにくい

SB:
   1 ステップで N 個の x, y を全部同時更新 → ベクトル並列で OK
   ↓
   GPU/FPGA で 100-1000 倍速になる
```

**収束の早さ**:
- SA: ランダム提案+受理判定の確率的探索(温度を下げながら遅延的)
- SB: 決定論的(あるいは bSB の弾道的)力学なので、**狙った方向に一直線**で進む

**ノイズ不要**:
- SA や CIM: 局所最適脱出にノイズが必須
- SB: ハミルトン力学の **エルゴード性**(位相空間を均一に動き回る性質)で同等以上の探索

## ステップ 10:CIM との対比(復習)

| 項目 | CIM | SB |
|---|---|---|
| 自由度 | c_i のみ(1自由度) | x_i + y_i(2自由度) |
| 力学 | **散逸 + 確率**(ASE ノイズが駆動) | **保存 + 決定論**(Hamilton 力学) |
| 「冷却」相当 | ポンプ P_p の増加 | 分岐パラメータ a(t) の増加 |
| 確率性の出所 | **毎ステップ N_I ガウス雑音** | **初期 x,y のみ**(あとは決定論) |
| 飽和機構 | ゲイン飽和 g_0(1−γI_in) | Kerr(aSB) / 反射壁(bSB/dSB) |
| 局所最適脱出 | ノイズ | **エルゴード性 + dSB のトンネル** |
| 並列性 | MFB 行列ベクトル積 | x, y の独立更新で完全並列 |

スローガン:
- **CIM** = 確率的ランジュバン力学(光ファイバ内の物理現象)
- **SB**  = 決定論的ハミルトン力学(計算機内の純粋数値計算)

## 重要なパラメータ

| パラメータ | 役割 | 推奨値 |
|---|---|---|
| `a_0` | 分岐パラメータの終端値(ポテンシャル深さ) | 1.0 |
| `a(t)` | 分岐パラメータの軌道 | `(t/T)·a_0`(線形ランプ) |
| `c_0` | 結合強度(正規化定数) | `0.5·√((N-1) / Σ J_ij²)` |
| `Δt` | 時間刻み | 0.5–1.0 |
| `N_step` | 総ステップ数 | 1000 程度 |
| `K` | Kerr 係数(aSB のみ) | 1.0 |
| `init_scale` | 初期 x, y の振幅 | 0.1 |
| `γ` | 加熱率(HbSB/HdSB のみ) | HbSB:0.5、HdSB:0.06 |

### c_0 の意味

線形不安定化条件 `a(t_c) = a_0 - c_0·λ_max(J)` から、c_0 は **「結合行列の固有値スケールを a_0 のスケールに合わせる正規化定数」**。大きすぎると分岐が早く・小さすぎると遅い。経験式 `c_0 = 0.5·√((N-1)/Σ J²)` は典型的な固有値スケールを ~1 に揃えるためのもの。

---

## まとめ:1分で説明できる SB

> N 個のボールを「ポテンシャルが時間とともに分裂する」ステージ上で転がす。最初は谷が 1 つだから全部原点にいるが、ステージの形が変わって谷が 2 つに分裂する瞬間、各ボールは結合 J_ij で引き合いながら左右どっちかに転がる。最後にボールが左右どっちにいるかでスピン ±1 を読み取れば、Ising 基底状態の良い近似解が得られる。Kerr の 4 乗項で谷を作るのが aSB、壁で谷を作るのが bSB、結合計算で符号だけ見るのが dSB。すべてエネルギー保存系を数値で解くだけなので、確率変数を使う SA や CIM より GPU/FPGA 並列化と相性が抜群。

実装は `modules/SB.py` の `_simulate_sb_batch` が 100 行程度。可変は分岐パラメータ a(t)、結合定数 c0、時間刻み Δt の 3 つだけ — シンプルさが SB が現代の Ising マシン主流になった理由です。
